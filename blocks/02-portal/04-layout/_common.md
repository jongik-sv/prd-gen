## 배경

전통적인 웹 애플리케이션은 한 번에 하나의 화면만 보여주는 단일 문서 인터페이스(SDI)를 사용한다. 하지만 산업 현장에서는 생산 주문 목록을 보면서 주문 상세를 확인하고, 동시에 품질 데이터를 모니터링하는 등 **다중 화면 동시 작업**이 필수적이다.

기존 C/S 프로그램에서는 MDI(Multiple Document Interface)로 자식 창을 자유롭게 배치했지만, 웹 환경에서는 이를 탭(TDI), 분할 뷰(Split View), 도킹(Docking) 등의 방식으로 대체한다.

### 기존 MDI → 웹 대응표

| 기존 MDI (C/S) | 웹 포털 대응 |
|---------------|-------------|
| 자식 창 열기 | 새 탭 열기 |
| 자식 창 타일 배치 | Split View (분할 뷰) |
| 자식 창 최소화 | 탭으로 존재 (항상 접근 가능) |
| 자식 창 드래그 이동 | 탭 드래그로 패널 간 이동 |
| 독립 창으로 분리 | Pop-out (새 브라우저 창) |
| 전체 화면 배치 저장 | 워크스페이스 저장 |

### 레이아웃은 재귀적 트리 구조로 표현

어떤 레이아웃 수준을 선택하든, 내부적으로 레이아웃은 **트리 구조**로 직렬화된다. 각 노드는 "리프(단일 패널)"이거나 "분할(두 자식으로 나뉜 패널)"이며, 이를 재귀적으로 조합하면 어떤 레이아웃이든 표현할 수 있다.

### 선택지 요약

| 선택지 | 동시 가시 화면 수 | 구현 복잡도 | 적합 대상 |
|--------|----------------|-----------|----------|
| 단일 화면 (SDI) | 1개 (전환 시 이전 화면 소실) | 매우 낮음 | 단순 관리 도구, 대시보드 |
| 순수 탭 (TDI) | 1개 (탭으로 상태 유지) | 낮음 | 일반 웹앱, 단순 업무 |
| 탭 + 분할 뷰 | 2~4개 (최대 2×2) | 중간 | MES, ERP 등 산업용 시스템 |
| 자유 도킹 | 무제한 | 매우 높음 | 트레이딩 터미널, IDE급 |
