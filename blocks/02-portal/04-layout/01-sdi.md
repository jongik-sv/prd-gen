## 개요

SDI(Single Document Interface)는 가장 전통적인 웹 애플리케이션 구조로, URL 경로에 따라 하나의 화면만 콘텐츠 영역에 표시한다. React Router, Vue Router 등 일반적인 SPA 라우터의 기본 동작이 이에 해당한다.

탭 시스템이 없으므로, 사용자가 다른 메뉴를 클릭하면 현재 화면이 unmount되고 새 화면이 mount된다. 이전 화면의 상태(폼 입력, 스크롤 위치, 필터 설정 등)는 별도 캐싱 로직이 없으면 소실된다.

## 레이아웃 구조

```
┌──────────────────────────────────────────────┐
│  Header  [통합검색] [알림] [프로필]            │
├─────┬────────────────────────────────────────┤
│     │  Content Area                          │
│  S  │  ┌────────────────────────────────┐    │
│  i  │  │                                │    │
│  d  │  │   현재 URL에 해당하는            │    │
│  e  │  │   단일 Remote App 표시           │    │
│  b  │  │                                │    │
│  a  │  │   (이전 화면은 unmount됨)        │    │
│  r  │  │                                │    │
│     │  └────────────────────────────────┘    │
└─────┴────────────────────────────────────────┘
```

## 화면 전환 동작

1. 사용자가 사이드바 메뉴 클릭
2. URL 변경 (예: `/project-a/dashboard` → `/project-a/orders`)
3. 현재 Remote App unmount
4. 새 Remote App mount (또는 같은 앱의 다른 경로로 전환)
5. 이전 화면의 상태는 소실 (별도 캐싱 없으면)

## 상태 보존 보완책

SDI의 가장 큰 단점인 상태 소실을 부분적으로 보완할 수 있는 방법:

| 보완책 | 설명 | 복원 범위 |
|--------|------|----------|
| URL 파라미터 | 필터, 정렬 조건을 URL 쿼리 파라미터에 반영 | 필터/정렬 |
| SessionStorage | unmount 시 상태를 SessionStorage에 저장, 재진입 시 복원 | 폼 입력, 스크롤 |
| 전역 상태 (Redux 등) | 앱 수준 상태 관리에 데이터 캐싱 | 로딩된 데이터 |
| Keep-Alive (Vue) | Vue의 keep-alive 컴포넌트로 상태 보존 | 컴포넌트 전체 |

이 보완책들을 적용하면 SDI에서도 어느 정도의 상태 보존이 가능하지만, TDI(탭)처럼 완벽한 보존은 아니다.

## TDI와의 비교

| 항목 | SDI (단일 화면) | TDI (탭) |
|------|---------------|---------|
| 동시 화면 수 | 1개 | 여러 개 (탭 수만큼) |
| 상태 보존 | 기본적으로 소실 | 탭이 열려있는 한 유지 |
| 메모리 사용 | 최소 (1개 앱만) | 탭 수에 비례 |
| URL 모델 | URL = 현재 화면 | URL = 활성 탭 |
| 구현 복잡도 | 매우 낮음 | 중간 |
| 화면 전환 속도 | 매번 mount/unmount | 숨기기/보이기 (즉각적) |

## 제약 사항 및 고려사항

1. **상태 소실**: 폼 작성 중 다른 화면으로 갔다 돌아오면 입력 내용이 사라짐 → 사용자 불만의 주요 원인
2. **동시 작업 불가**: 두 화면을 비교하거나 참조하면서 작업하는 것이 불가능
3. **전환 비용**: 빈번한 화면 전환이 필요한 업무에서 매번 로딩 지연이 누적됨
4. **산업 현장 부적합**: MES/ERP 운영자에게는 치명적 한계 (동시 3~5개 화면이 필수)
5. **마이그레이션**: 기존 C/S MDI 환경 사용자에게 SDI는 심각한 기능 퇴보로 인식됨
