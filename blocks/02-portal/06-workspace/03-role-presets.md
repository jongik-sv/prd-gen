## 개요

역할별 프리셋 전략은 관리자(또는 시스템)가 역할(직무)별로 미리 정의된 워크스페이스를 제공하는 방식이다. 사용자가 처음 로그인하면 자신의 역할에 맞는 프리셋이 자동으로 적용되어, 별도 설정 없이 최적 환경에서 작업을 시작할 수 있다.

이 전략의 핵심 가치는 **"역할 기반 표준화"**이다. 같은 업무를 하는 사람들이 같은 화면 구성을 사용함으로써 교육, 지원, 업무 프로세스가 일관되게 운영된다.

## 프리셋 구조

### 역할-프리셋 매핑 예시

| 역할 | 프리셋 이름 | 탭 구성 | 레이아웃 |
|------|-----------|--------|---------|
| 생산관리자 | "생산 관제" | 생산현황, 작업지시, 품질검사, 설비상태 | 좌측 목록 + 우측 상세 (2분할) |
| 품질관리자 | "품질 관리" | 품질검사, 불량이력, SPC차트 | 상단 차트 + 하단 목록 (2분할) |
| 물류담당 | "물류 관리" | 입고현황, 재고현황, 출고지시 | 3탭 순차 |
| 경영진 | "대시보드" | 종합대시보드 | 단일 화면 |

### 사용자 커스터마이징

프리셋은 기본값이지 강제값이 아니다:

1. 사용자가 프리셋 기반 위에서 탭 추가/제거, 레이아웃 변경 가능
2. 변경된 환경을 "내 워크스페이스"로 저장 가능
3. "기본 레이아웃으로 복원" 기능으로 프리셋으로 쉽게 초기화
4. 관리자 프리셋과 사용자 커스텀 워크스페이스가 공존

## 프리셋 관리

### 관리자 기능

| 기능 | 설명 |
|------|------|
| 프리셋 생성 | 관리자가 직접 탭과 레이아웃을 구성하여 프리셋으로 저장 |
| 역할 매핑 | 생성한 프리셋을 하나 이상의 역할에 연결 |
| 프리셋 수정 | 기존 프리셋의 탭/레이아웃 변경 |
| 적용 정책 | 프리셋 변경 시 기존 사용자에게 어떻게 적용할지 결정 |

### 프리셋 변경 적용 정책

관리자가 프리셋을 수정했을 때, 기존 사용자에게 어떻게 반영할지:

| 정책 | 동작 | 적합 상황 |
|------|------|----------|
| **알림만** | "프리셋이 업데이트되었습니다. 적용하시겠습니까?" 안내 | 사용자 자율 중시 |
| **미커스터마이징 사용자만 적용** | 프리셋을 변경하지 않은 사용자에게만 자동 적용 | 균형 잡힌 접근 |
| **강제 적용** | 모든 사용자의 워크스페이스를 새 프리셋으로 덮어쓰기 | 보안/규정 요구 |

## 다중 역할 처리

한 사용자가 여러 역할을 가질 수 있는 경우:

| 시나리오 | 처리 방식 |
|---------|----------|
| 단일 역할 | 해당 역할의 프리셋 자동 적용 |
| 다중 역할 | 주 역할(primary role)의 프리셋 적용, 나머지 프리셋은 선택 가능 목록에 표시 |
| 역할 없음 | 시스템 기본 프리셋 (대시보드 1개 탭) 적용 |

## 제약 사항 및 고려사항

1. **역할 체계 의존**: 조직의 역할 정의가 명확해야 효과적이며, 역할이 모호하면 프리셋이 맞지 않음
2. **관리 부담**: 역할별 프리셋을 설계하고 변경 사항을 반영하는 지속적인 관리가 필요
3. **앱 변경 연동**: Remote App이 추가/삭제/변경되면 관련 프리셋도 업데이트해야 함
4. **관리자 UI**: 프리셋을 시각적으로 편집하는 관리자 도구가 추가로 필요
