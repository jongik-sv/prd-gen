## 개요

사용자 저장 워크스페이스는 현재 열려있는 탭 목록, 레이아웃 분할 설정, 패널 링크 등을 사용자 계정에 연결된 스냅샷으로 서버에 저장하는 기능이다. 저장된 워크스페이스를 선택하면 해당 환경이 자동으로 복원된다.

### 워크스페이스가 저장하는 정보

| 항목 | 설명 |
|------|------|
| **탭 목록** | 열려있는 탭의 앱 ID, 경로, 파라미터, 제목, 순서 |
| **레이아웃** | 패널 분할 트리 구조 (방향, 비율, 패널별 탭 배치) |
| **패널 링크** | 마스터-디테일 연동 설정 |
| **Pop-out** | Pop-out 창의 앱, 경로, 창 위치/크기 |
| **메타데이터** | 이름, 생성 시각, 마지막 수정 시각, 기본 워크스페이스 여부 |

## 핵심 기능

### 저장

- 현재 상태를 새 이름으로 저장 (Save As)
- 기존 워크스페이스를 현재 상태로 덮어쓰기 (Save)
- 최대 저장 수 제한 (예: 사용자당 10개)

### 복원

- 워크스페이스 선택 시 현재 탭을 모두 닫고 저장된 환경으로 전환
- 기본 워크스페이스 지정 시 로그인 후 자동 복원
- 복원 시 권한이 없는 앱의 탭은 건너뛰고 안내 표시

### 관리

- 워크스페이스 이름 변경, 삭제
- 기본 워크스페이스 지정/해제
- 목록 조회 (이름, 탭 수, 마지막 사용 시각)

## 저장 위치

| 방식 | 크로스 디바이스 | 용량 제한 | 서버 의존 |
|------|-------------|----------|----------|
| **서버 API** | O (어디서든 복원) | 서버 정책에 따름 | O |
| **LocalStorage** | X (해당 브라우저만) | ~5MB | X |
| **IndexedDB** | X (해당 브라우저만) | ~50MB+ | X |

권장: 서버 API로 저장하여 다른 PC/브라우저에서도 복원 가능하게 한다. LocalStorage는 오프라인 fallback용으로만 사용한다.

## 자동 저장

마지막 작업 상태를 자동으로 저장하여, 브라우저를 닫았다가 다시 열어도 이전 환경을 복원할 수 있다:

- 탭 열기/닫기, 레이아웃 변경 시 자동으로 "마지막 세션" 워크스페이스에 저장 (디바운스 적용)
- 로그인 시 "마지막 세션으로 복원하시겠습니까?" 프롬프트 또는 자동 복원 설정

## 제약 사항 및 고려사항

1. **신규 사용자 경험**: 처음 로그인한 사용자에게 저장된 워크스페이스가 없으므로 빈 상태에서 시작
2. **앱 변경 대응**: Remote App이 삭제되거나 경로가 변경되면 저장된 워크스페이스의 해당 탭이 깨짐 → 유효성 검증 필요
3. **교육/지원**: 사용자마다 다른 환경을 사용하면 "내 화면에서는 이렇게 보이는데..."식의 지원 요청이 증가
4. **백업**: 사용자가 실수로 워크스페이스를 삭제하면 복구 방법이 없음 (서버 측 soft delete 고려)
