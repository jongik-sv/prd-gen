## 배경

산업 현장에서 사용자는 매일 같은 화면 구성으로 작업한다. MES 생산관리자는 "생산현황 + 작업지시 + 품질검사"를, 물류담당자는 "입고현황 + 재고현황 + 출고지시"를 항상 열어둔다. 이런 반복적인 화면 구성을 자동화하는 것이 워크스페이스이다.

### 워크스페이스가 저장하는 정보

- 열려있는 탭 목록 (앱 ID, 경로, 파라미터, 순서)
- 레이아웃 트리 (패널 분할 구조, 비율)
- 패널 링크 설정 (마스터-디테일 연동)
- Pop-out 창 정보 (앱, 경로, 창 위치/크기)

### 선택지 요약

| 선택지 | 기능 범위 | 구현 복잡도 |
|--------|----------|-----------|
| 미지원 | 워크스페이스 없음, 매번 수동 구성 | 없음 |
| 사용자 저장만 | 사용자가 자신의 워크스페이스를 저장·복원 | 낮음 |
| 역할별 프리셋 | 관리자가 역할별 기본 워크스페이스를 정의 | 중간 |
| 전체 (프리셋 + 관리자 배포) | 프리셋 + 관리자 배포 + 사용자 커스터마이징 + 크로스 디바이스 복원 | 높음 |
