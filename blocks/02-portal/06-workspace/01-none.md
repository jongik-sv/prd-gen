## 개요

워크스페이스 미지원은 의도적으로 워크스페이스 기능을 포함하지 않는 선택이다. 사용자는 포털에 접속할 때마다 빈 상태에서 시작하며, 사이드바 메뉴를 클릭하여 필요한 탭을 직접 열고 레이아웃을 구성한다.

이 방식은 포털이 단순한 구조(탭 1~2개, 분할 없음)일 때 합리적이다. 하지만 MES/ERP처럼 5개 이상의 탭을 열고 Split View를 구성하는 환경에서는 매일 아침 같은 설정을 반복하는 것이 비효율적이다.

## 기본 동작

### 로그인 시

1. Shell App 로딩
2. 빈 콘텐츠 영역 표시 (또는 기본 대시보드 한 개 자동 열림)
3. 사용자가 사이드바에서 필요한 메뉴를 클릭하여 탭 생성

### 새로고침 / 브라우저 재시작

- 모든 탭 상태 소실
- 현재 URL의 화면만 단일 탭으로 복원
- Split View 레이아웃 소실

### 다른 PC에서 로그인

- 완전히 새로운 시작
- 이전 환경의 정보가 전혀 없음

## 최소한의 보완책

워크스페이스 없이도 사용성을 일부 개선할 수 있는 방법:

| 보완책 | 설명 | 효과 |
|--------|------|------|
| 기본 홈 화면 | 로그인 시 대시보드 자동 열기 | 빈 화면 방지 |
| URL 기반 복원 | 현재 활성 탭의 URL을 브라우저에 반영, 새로고침 시 해당 탭 복원 | 단일 탭 복원 |
| SessionStorage | 브라우저 탭 내에서 탭 목록을 임시 보관, 새로고침 시 복원 | 탭 목록 임시 복원 (세션 한정) |
| 즐겨찾기 메뉴 | 자주 쓰는 메뉴를 즐겨찾기로 등록하여 빠른 접근 | 탭 열기 단축 |

## 제약 사항 및 고려사항

1. **사용자 불만**: 복잡한 작업 환경을 매번 재구성해야 하므로, 사용자 만족도 저하 가능
2. **생산성 손실**: 매일 5분씩 환경을 구성하면 연간 20시간 이상의 손실
3. **온보딩 어려움**: 신규 사용자가 어떤 화면을 열어야 하는지 안내할 방법이 없음
4. **추후 마이그레이션**: 나중에 워크스페이스를 도입할 때 기존 사용자의 습관과 충돌할 수 있음
