## 배경

하나의 브라우저 탭에서 여러 Remote App이 동시에 실행될 때, 다음과 같은 보안 위협이 존재한다:

| 위협 유형 | 설명 | 위험도 |
|----------|------|--------|
| **전역 변수 오염** | 한 앱이 `window` 객체에 쓴 값을 다른 앱이 읽거나 덮어씀 | 높음 |
| **DOM 조작** | 한 앱이 다른 앱의 DOM 요소를 수정하거나 이벤트를 가로챔 | 높음 |
| **CSS 오염** | 한 앱의 전역 CSS가 다른 앱의 스타일을 망가뜨림 | 중간 |
| **토큰 탈취** | 악의적인 앱이 공유 컨텍스트에서 인증 토큰을 가로챔 | 매우 높음 |
| **XSS 전파** | 한 앱의 XSS 취약점이 같은 컨텍스트의 다른 앱에도 영향 | 매우 높음 |

### 격리 메커니즘 비교

| 방식 | JS 격리 | CSS 격리 | DOM 격리 | 성능 영향 | 통신 방식 |
|------|---------|---------|---------|----------|----------|
| 동일 컨텍스트 (없음) | X | X | X | 없음 | 직접 호출 |
| Shadow DOM | X | O | 부분 | 경미 | 직접 호출 |
| Web Worker | O | - | DOM 불가 | 경미 | postMessage |
| iframe (동일 출처) | O | O | O | 중간 | postMessage |
| iframe (교차 출처) | O | O | O | 큼 | postMessage |

### 선택지 요약

| 선택지 | 핵심 전략 | 적용 대상 |
|--------|----------|----------|
| 동일 컨텍스트 | 격리 없이 코드 리뷰·CI/CD로 신뢰 확보 | 모든 앱이 내부 팀 소유 |
| iframe 격리 | 모든 외부 앱을 iframe sandbox로 격리 | 모든 앱이 외부/서드파티 |
| 2-tier (내부/외부 분리) | 내부 앱은 동일 컨텍스트, 외부 앱은 iframe | 내부 + 외부 앱 혼재 |
