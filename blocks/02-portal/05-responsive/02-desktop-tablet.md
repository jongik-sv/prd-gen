## 개요

데스크톱 + 태블릿 반응형 정책은 최소 뷰포트 768px 이상을 공식 지원하는 전략이다. 데스크톱에서는 모든 기능을 완전하게 사용하고, 태블릿에서는 레이아웃이 적응되어 핵심 기능을 터치로 조작할 수 있다.

이 전략의 핵심은 **"데스크톱 최적, 태블릿 적응"**이다. 태블릿을 위해 데스크톱 경험을 희생하지 않으며, 태블릿에서는 가장 중요한 기능이 사용 가능한 수준으로 적응한다.

## 뷰포트별 동작

### 데스크톱 (1280px+)

- 사이드바 항상 펼침
- Split View 최대 4분할 (1920px+) 또는 3분할 (1280~1919px)
- 마우스 + 키보드 인터랙션 최적화
- Pop-out(새 창 분리) 지원
- 키보드 단축키 전체 지원

### 태블릿 (768~1279px)

| 요소 | 데스크톱 | 태블릿 적응 |
|------|---------|-----------|
| 사이드바 | 항상 펼침 | 접힘 (햄버거 메뉴로 토글) |
| Split View | 최대 4분할 | 최대 좌우 2분할만 허용 |
| Pop-out | 새 브라우저 창 | 미지원 (태블릿 OS 제약) |
| 드래그 앤 드롭 | 마우스 기반 | 터치 롱프레스 + 드래그 |
| 컨텍스트 메뉴 | 우클릭 | 롱프레스 |
| 툴팁 | 마우스 호버 | 미표시 또는 탭 시 표시 |
| 키보드 단축키 | 전체 지원 | 미지원 (외부 키보드 제외) |

### 패널 축소 규칙

데스크톱에서 4분할로 작업하던 사용자가 태블릿으로 전환하면:

1. 현재 활성 패널 2개만 좌우 분할로 유지
2. 나머지 패널은 탭으로 전환 (convert-to-tab)
3. 사용자에게 "화면 크기에 맞게 레이아웃이 조정되었습니다" 안내
4. 원래 레이아웃은 워크스페이스에 저장되어 있으므로, 데스크톱 복귀 시 복원 가능

## 터치 최적화

태블릿 지원을 위한 터치 인터랙션 최적화:

| 영역 | 최적화 내용 |
|------|-----------|
| 터치 타겟 크기 | 최소 44×44px (Apple HIG / Material Design 기준) |
| 탭 바 | 탭 높이 확대, 터치로 드래그 순서 변경 |
| 분할선 | 터치 영역 확대 (8px → 20px), 드래그 피드백 강화 |
| 스크롤 | 관성 스크롤, 스와이프 제스처 지원 |
| 테이블 | 터치 스크롤, 행 높이 확대 |

## 테스트 전략

| 카테고리 | 데스크톱 | 태블릿 |
|---------|---------|--------|
| 브라우저 | Chrome, Edge, Firefox | Safari (iPad), Chrome (Android Tablet) |
| 해상도 | 1920×1080, 1366×768 | 1024×768, 1194×834 |
| 인터랙션 | 마우스 + 키보드 | 터치 + (선택) 키보드 |
| 회전 | N/A | 가로/세로 전환 |

## 제약 사항 및 고려사항

1. **모바일 미지원**: 스마트폰에서는 사용 불가하며, 급한 확인이 필요한 경우 별도 모바일 앱이 필요
2. **태블릿 성능**: 태블릿의 하드웨어 성능이 데스크톱 대비 낮아, 대량 데이터 렌더링 시 성능 저하 가능
3. **테스트 비용**: 터치 인터랙션 테스트, 태블릿 브라우저 호환성 테스트 추가 필요
4. **태블릿 전용 버그**: 가로/세로 회전 시 레이아웃 깨짐, 소프트 키보드 표시 시 뷰포트 변화 등 태블릿 특유의 엣지 케이스
