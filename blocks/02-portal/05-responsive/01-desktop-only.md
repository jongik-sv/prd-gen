## 개요

데스크톱 전용 정책은 최소 뷰포트 1280px(일반적으로 1920px 권장) 이상의 환경만 공식 지원하는 전략이다. 반응형 레이아웃 구현을 생략하고, 고정 레이아웃 또는 넓은 화면 기반 레이아웃에 집중한다.

이 방식은 "제한된 자원으로 최고의 데스크톱 경험을 제공한다"는 실용적 판단에 기반한다. 반응형 대응에 투입될 리소스를 핵심 업무 기능에 집중할 수 있다.

## 지원 환경

### 공식 지원

| 환경 | 해상도 | Split View | 비고 |
|------|--------|-----------|------|
| 대형 모니터 (27"+) | 2560×1440+ | 4분할 가능 | 최적 환경 |
| 일반 모니터 (24") | 1920×1080 | 4분할 가능 | 권장 환경 |
| 소형 모니터 (21") | 1680×1050 | 3분할 가능 | 지원 |
| 노트북 (15") | 1366×768+ | 2분할 가능 | 최소 지원 |

### 미지원 (경고 표시)

| 환경 | 해상도 | 동작 |
|------|--------|------|
| 소형 노트북 (13") | 1280×800 미만 | "데스크톱 환경에서 사용하세요" 안내 표시 |
| 태블릿 | 1024×768 | 동일 안내 표시 |
| 모바일 | 다양 | 동일 안내 표시 |

## 설계 단순화 효과

반응형을 고려하지 않으면 다음 영역의 복잡도가 크게 줄어든다:

| 영역 | 반응형 시 필요 | 데스크톱 전용 시 |
|------|-------------|---------------|
| CSS 미디어 쿼리 | 다중 브레이크포인트 관리 | 불필요 |
| 터치 이벤트 | 터치/스와이프 대응 | 불필요 (마우스만) |
| 사이드바 | 펼침/접힘/하단탭 3가지 모드 | 항상 펼침 |
| Split View | 뷰포트별 패널 수 제한 | 항상 최대 4분할 가능 |
| 테스트 | 다중 디바이스/뷰포트 테스트 | 주요 해상도 2~3개만 |
| 폰트/간격 | 뷰포트별 유동 크기 | 고정 크기 |

## 향후 확장 가능성

데스크톱 전용으로 시작하더라도, 향후 태블릿 지원을 추가해야 할 경우를 대비한 최소한의 설계 원칙:

1. **상대 단위 사용**: 가능한 `rem`, `%`를 사용하여 절대값 의존 최소화
2. **컴포넌트 분리**: 레이아웃과 콘텐츠를 분리하여, 레이아웃만 교체 가능하게 설계
3. **Design Token**: 뷰포트별 토큰 오버라이드가 가능한 구조로 토큰 설계
4. **최소 너비 기준**: 1280px 미만에서도 가로 스크롤 없이 핵심 기능은 사용 가능하도록

## 제약 사항 및 고려사항

1. **접근성 제한**: 태블릿/모바일 사용자가 포털에 접근할 수 없음
2. **현장 사용**: 생산 현장에서 태블릿으로 데이터를 확인하는 시나리오가 점점 일반화되고 있음
3. **원격 근무**: 재택 근무자의 개인 장비가 소형 노트북일 수 있음
4. **규제 요구**: 일부 접근성 규제(WCAG)에서 다양한 디바이스 지원을 요구할 수 있음
