# PRD Generator - 블록 문서 작성 프롬프트

## 용도

이 프롬프트는 브레인스토밍 요약 문서(summary/)로부터 PRD 블록 문서(blocks/)를 생성할 때 사용한다.
다른 파트(Part 1, Part 3 등)에도 동일한 패턴으로 적용할 수 있다.

---

## 프롬프트

```
# 블록 문서 작성 지침

## 입력
- 소스: `summary/brainstorming-part{N}-{topic}-summary.md`
- 출력: `blocks/{NN}-{part-name}/{category}/{file}.md`

## 디렉토리 구조 규칙
- 카테고리 디렉토리: `{NN}-{category-name}/` (번호 + 이름)
- 공통 배경 파일: `_common.md` (카테고리별 1개)
- 옵션 블록 파일: `{NN}-{option-name}.md` (번호 + 이름)

## _common.md 형식 (카테고리별 배경 문서)

---
category: [카테고리 ID]
label: [카테고리 표시명]
summary: [카테고리 전체를 설명하는 한 줄 요약]
context:
  - [배경 설명 1]
  - [배경 설명 2]
dependencies:
  - [관련 카테고리 ID]
---

## 배경
[왜 이 결정이 필요한지, 어떤 문제를 해결하는지 서술]

### [핵심 개념 또는 비교표]
[표 또는 개념 설명]

### 선택지 요약
[모든 옵션의 비교표]


## 옵션 블록 .md 형식 (개별 선택지)

---
id: [옵션 ID]
category: [카테고리 ID]
label: [표시명]
summary: [한 줄 요약]
pros:
  - [장점 - 구체적으로, 웹 UI에서 사용자가 선택 근거로 삼을 수 있도록]
cons:
  - [단점 - 구체적으로]
bestFor:
  - [이 옵션이 적합한 시스템/상황]
---

## 개요
[이 옵션이 무엇인지, 핵심 가치는 무엇인지 2~3문단으로 서술]

## [상세 챕터 1] (예: 아키텍처, 구성 요소, 동작 방식)
[해당 옵션의 핵심 구조를 표, 다이어그램(ASCII art), 자연어로 설명]

## [상세 챕터 2] (예: 주요 기능, 전략, 비교)
[세부 동작이나 하위 선택지를 표와 설명으로 정리]

## [상세 챕터 N]
[필요한 만큼 챕터 추가]

## 성능 특성 (해당 시)
[성능 관련 비교표]

## 제약 사항 및 고려사항
[번호 매긴 목록으로 제약사항 정리]


## 작성 원칙

1. **코드 없이 설명**: 코드 스니펫(TypeScript, JavaScript, CSS 등)은 포함하지 않는다.
   코드가 설명하던 내용은 자연어, 표, ASCII 다이어그램으로 대체한다.
   코드는 TRD(기술 설계서) 단계에서 다룬다.

2. **충실한 설명**: 웹 UI에서 사용자가 각 옵션을 읽고 선택할 수 있도록 충분히 상세하게 작성한다.
   - _common.md: 해당 카테고리의 배경과 왜 결정이 필요한지를 명확히 설명
   - 옵션 블록: 개요 → 상세 챕터들 → 제약사항 순서로 구조화

3. **챕터 분리**: 소개(개요)와 상세 내용은 반드시 별도 챕터로 분리한다.
   한 챕터에 모든 내용을 몰아넣지 않는다.

4. **표 적극 활용**: 비교, 분류, 옵션 나열은 가능한 표(Markdown table)로 정리한다.

5. **ASCII 다이어그램**: 아키텍처 구조, 계층 구조, 흐름도는 ASCII art로 표현한다.

6. **한국어 작성**: 모든 설명은 한국어로 작성한다. 기술 용어(Module Federation, WebSocket 등)는 원어 그대로 사용.

7. **파일명 번호**: 옵션 블록 파일명에 번호를 포함한다 (01-, 02-...).
   _common.md는 번호 없이 유지한다.

## 작업 순서

1. 소스 summary 파일을 읽고 카테고리와 옵션을 파악
2. 카테고리 디렉토리 생성
3. 각 카테고리의 _common.md 작성
4. 각 옵션 블록 파일을 번호 포함하여 작성
5. 전체 파일 목록 확인

## 예시 디렉토리 구조

blocks/02-portal/
├── 01-mf-integration/
│   ├── _common.md
│   ├── 01-module-federation.md
│   ├── 02-single-spa.md
│   ├── 03-iframe.md
│   └── 04-hybrid.md
├── 02-security-isolation/
│   ├── _common.md
│   ├── 01-same-context.md
│   ├── 02-iframe-sandbox.md
│   └── 03-2-tier.md
└── ...
```

## 사용법

새로운 파트의 블록 문서를 작성할 때:

1. 이 프롬프트를 Claude에게 전달
2. 소스 파일 경로와 출력 경로를 지정
3. 작성할 카테고리와 옵션 목록을 정리하여 전달

예시 요청:
```
위 프롬프트의 지침에 따라 블록 문서를 작성해줘.

소스: summary/brainstorming-part3-backend-summary.md
출력: blocks/03-backend/
카테고리: 01-api-gateway, 02-auth, 03-database, ...
```
